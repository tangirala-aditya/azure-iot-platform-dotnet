    Optional Kubernetes Hardening

    The Azure Application Gateway ("App Gateway") and Application Gateway Ingress Controller ("AGIC") can be used to add additional security and performance improvements to the Serenity Platform. Although Microsoft recommends it, it is currently an optional add-on, primarily because of the additional costs.

    This document provides details about AGIC so that you can decide if you want to apply it to your platform's configuration.

    Overview

    The Application Gateway Ingress Controller (AGIC) is a Kubernetes application, which makes it possible for Azure Kubernetes Service (AKS) customers to leverage Azure's native Application Gateway L7 load-balancer to expose cloud software to the Internet. By default, Serenity's Kubernetes cluster is configured using the "Basic Networking" model.

    The optional "Advanced Networking" can be used in an AKS Cluster so that Nodes use the Azure Container Networking Interface (CNI) so that K8s Nodes are bound to specific IP addresses instead of broadcasting.

    Benefits

    AGIC offers the following advanages:

    • Security Hardening
      Serenity APIs are protected by a native Web Appliation Firewall which reduces vulnerabilities like Denial of Service Attacks
    • Role Based Access Control (RBAC)
      Service and User Accounts can be use to restrict access to AKS endpoints
    • Increased Performance
      AGIC removes two extra service hops which consequently improves application performance by approximately 30 milliseconds.

    Costs

    In general, AGIC increases platform costs by approximately $300 per month. AGIC does not have to be applied to all environments. It can, for example, be applied to just production environments to reduce platform costs.
